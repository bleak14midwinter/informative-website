Underneath you can see my finished contact form after I styled it in CSS

Independent research - Long word breaking

When playing around with the developer tools on Google Chrome by resizing the webpage to emulate small devices, testing its responsiveness, I found a problem. Long words, or really only long URLs would not make the webpage to resize properly, as the text would be too long and uninterupted for an entire width of a line, making it not a responsive web design. This is seen in the image on the right.

After reading about the CSS word-break property on MDN Web Docs, I found that body { word-break: break-word; } fixed the issue. This cause long URLs to be broken up onto different lines, as seen in the picture on the left, causing my webpage to become responsive on devices with small screens.

Week 8 - CSS flexbox and multi-column responsive layout

Posted on November 25, 2023

This week I added some content to my tutorial page on study tips during lockdown, and then tried 2 new ways of laying out content in a responsive web page for the tutorial page, the CSS3 flexbox layout and the CSS multi-column layout. My tutorial page has content with multiple steps, which I hope to layout using the 'card' interface design which you can find out more about here: https://www.smashingmagazine.com/2016/10/designing-card-based-user-interfaces/.
Underneath is my tutorial page with the new content using the same grid layout as this learning journal page.

Flexbox layout

I used the CSS flexbox property to lay out my tutorial steps as flexible cards in columns, and made the images flexible. In the CSS I used media queries to reduce the number of columns as the viewport width decreases. You can find out more about the flexible box layout here.
Below is a picture of my tutorial page with the new content laid out using flexbox.

Multi-column layout

Like the flexbox, I used media queries in the CSS do determine how many columns will be used for different screen widths. Content in the tutorial are laid out in article elements with images that are floated so that text wraps around them. MDN - Using CSS multi-column layout provides more infomation on this layout I implemented in my tutorial page,
which you can see here underneath.

Comparison - Flexbox vs Multi-column

After trying out both the CSS flexbox layout and multi-column layout on my tutorial page with the same content, I came to the conclusion that I will continue to use the flexible box layout. This is becuase it suited my content more, as I had multiple steps which look best is they match the card interface design.

After adding more content to the tutorial page, I decided to use 3 different sized 'cards' with classes .card-small, .card-medium, .card-large. In the CSS, my media queries how many of each sized card can be adjacent for the device viewing the page.
For example, as seen on the right, for desktops and laptops the tutorial page has 4 columns of small cards at the top and a card using the entire width underneath.
This ease of flexibilty is another reason that I decided to a flexbox instead of multi-column layout.

Week 7 - Making media flexible

Posted on November 18, 2023

Clipped and flexible images

Underneath is an image with a size similar to a banner and with the property wrapper overflow set to hidden. This means that as the webpage is resized to a shorter width, part of the image will be removed, whilst remaining the same size.

To do this I embed the HTML for the image in a <header which has the class="img-clipped">. Then in the CSS i add the following rule: .img-clipped { overflow: hidden; }.

Now see the same image underneath which is now flexible, meaning that as the viewport is resized smaller, so does the image automatically.

I did this by embeding the same image in the HTML with no height or width attributes in the <img> element, which now has the class class="img-fluid". Then add the following CSS rule: .img-fluid { width: 100%; max-width: 900px; } with the max-width being the actual width of the image.

Here is a smaller image with this text being wrapped around it flexibly, where the image can resize. This is done using the float and clear properties in the CSS. First in the HTML the image is given the class class="img-wrap-left-small". Then the text that you want to be wraped around the image written inside <p> tags in the following line after the image. In the next line in the HTML you write <div class="clear"></div>, and finally add 2 rules in the CSS: .img-wrap-left-small { float: left; padding: 2%; width: 33%; } and .clear { clear: both; }.
The float and width properties can be altered or change in a new class, if you want the image to wrap on the right and/or make up a bigger fraction of the width. For example I made another class img-wrap-right-large which is used for the next wrapping image below.

Flexible videos

Underneath is a embedded video from YouTube which is in a flexible video container. This means that if the page if resized smaller, the video will shrink as well, whilst maintaining a 16:9 width:height ratio, benefiting the reader of the page.

To find the HTML that you copy into your webpage to embed a YouTube video, you first need find the video you want to use. Then you press the Share button underneath the video, which then brings up the option for you to press Embed. It will then show the HTML inside <iframe> tags which can be copied over.
Also notice how here I have used an image with the class img-wrap-right-large, where in the CSS I styled the image to be 2/3 width and wrap on the right instead.

Extra - background images

All elements in HTML, have a background property that using the CSS, can be styled. Images for decoration, not images for content in the page, can be put into the background of elements, instead of in the HTML using a img tag.

Images that are small can be repeated as tiles to form a pattern when they are put in the background of elements. I used this Stripe Generator tool to find a image that is currenty used as a tile for the background image of this <p> element, which this text is written in. I chose this small tile, which is only 54 x 45 pixels, because as it is repeating in the x and y axis, it creates a shooting star pattern
I did this by giving the above element the class: class = "background-tiled" and adding the following rule to the CSS: .background-tiled { background-image: url(http://www.stripegenerator.com/generators/generate_shared_stripe.php?stripe=3026); }.

Because I identified the image souce using its full URL as it hosted on another web server , as I fullback I also declered the property background-color: #666, because it may fail to retrieve the image from the external website.
*UPDATE* as explained below this image did fail so I used a non hosted background image: background-image: url(../images/stripe.png);.

You can turn of tiling using the property background-repeat: no-repeat; in the background of an element in CSS, if you want to use a single large image instead. You can also secify is you just want the background image of an element repeated vertically or horizontally by using either background-repeat: repeat-x; or background-repeat: repeat-y;.

A parallax effect can be created by fixing the position of a background image of an element, meaning that as the page content is scrolled up and down, the background image will remain stationery. As you can see I did this for this <p> element in which this text currently is, by giving it the class: class = "background-fixed" and adding the following rule to the CSS: .background-fixed { background-attachment: fixed; }. However when viewing my webapge on an iPhone 6, the image appeared to scroll as normal, meaning that this property and parallax effect won't work on all devices or browsers, and the image will remian to look like a normal one.

Check out this MDN page - https://developer.mozilla.org/en-US/docs/Web/CSS/background as it has more infomation on all of the CSS background properties.

Extra - Using a favicon

A favicon is a small picture that is shown in browser's address bar or tab, next to the title of your webpage, when it is visited, or may be shown if you favorite or bookmark the page. It can be a finishing touch to your website and is important for branding it, as this website explains: Web Usability: What Is Favicon and How It Improves Your Website.

These links will help you make you own favicon

  • http://tools.dynamicdrive.com/favicon/ - Favicons have there own permited file format, so this tool creates a favicon from an uploaded image.
  • http://www.favicon.cc/This tool allows you to draw your own favicon. I used colours I already used in my website for consistancy to make this favicon:

Once I made made my favicon, in the <head> of all 3 of my HTML documents, I put a link to the file with: <link rel="icon" type="image/ico" href="images/favicon.ico">. This website provides infomation on the different file types and sizes you can use: The Icon Handbook.

Error - Missing externally hosted images

When I loaded my webpage, I noticed that my background image that was hosted on an external website was not showing, even though I made sure that in the CSS the full URL to identify the image source was correct, and that the image could be found when I entered it's URL in my browser's address search.

When I opened the Chrome developer tools, I saw the warning message seen here to the left.

This meant the request to get it was upgraded from the insecure HTTP, to HTTPS because I loaded my webpage with HTTPS (which is done by default when typing my webpage's URL without specifying the protocol). So the image was not found because it can only be found with HTTP, by my testing in the URL seach bar. When I loaded my webpage specifying with only HTTP, the image as I expected was showing, because the request to find it was not automatically upgraded to HTTPS.

Becuase most visiters to my webpage won't specify the protocol, I decided to download the image and uploaded it to my images folder. I then identify the image souce from there, so that the image would always be displayed for everyone.

Independent research - Smooth scrolling

This page on w3schools gave me infomation on how to create a smooth scrolling effect with CSS. When I added the property scroll-behavior: smooth to the <html> element, smooth scrolling was enabled for the whole page. This meant that instead of a straight jump, when the user clicks on a link within my website (such as the navigation list), there is a smooth animation of scrolling to the selected position.
